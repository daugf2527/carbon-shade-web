## 一、核心成果

### 🎯 玩家不再一击秒杀

**问题**: Stage 2 完成时，玩家 attack1 打 grunt 造成 51080 伤害，一击秒杀 160 HP 敌人。

**根因**: `DamageFormula.ts` 将 `atkPower=1800` 直接当乘数，公式错误：
```
multiplier = statRatio(2.92) × atkPower(1800) × eleRatio(1.0) × defRatio(0.972) × ...
           = 5108
finalDamage = baseDamage(10) × 5108 = 51080  // 一击秒杀
```

**修复**: Phase A T-A.9 推倒重写 DamageFormula，`atkPower` 改为基数而非乘数：
```
finalDamage = (chr.growth.physicalAttack × level + weapon.physAtk) 
              × attack.damageBonus × weapon.damageScalePct × (1 - defRatio)
```

**结果**: attack1 → grunt 造成 **23 伤害**（160 → 137 HP），符合 DNF 真实战斗体验。

## 三、关键技术决策

### D8=C: 砍掉 default.json

**决策**: 删除 `src/data/manifest/actions/default.json`（38 个硬编码动作），全部改用 PVF 真值命名。

**影响**:
- ✅ 代码简化：不再维护两套命名（NormalBasic1 vs attack1）
- ✅ 真值驱动：所有动作数据来自 PVF，无脑补值
- ⚠️ 破坏性变更：Stage 2 所有引用 default.json 的测试需要修改

**实施**: T-A.5 + T-A.10 + T-A.12（修复 5 个 baseline fail）

### D9=B: reaction 公式降级

**决策**: Phase C 超时（1 周时间盒），采用 D9=B 降级路径：
- ✅ 反推文档骨架完成（T-C.1）
- ⚠️ 公式实装推迟到 Phase E 或后续 Stage

**原因**: 
- hitstun/launch 曲线在 DNF.exe C++ 二进制里，PVF 拿不到
- 需要大量实测 DNF 客户端行为 + 公式假设验证（T-C.2-7，预估 20+ 小时）

**影响**: ReactionResolver 接受真值参数但公式仍用 local_baseline stub

### D7=C: 编译时嵌入 bundle

**决策**: swordman-truth.json (248KB) 编译时嵌入 bundle，不做 lazy load。

**理由**:
- Bundle 主包 1660 KB（~1.6MB），truth 数据占 ~15%
- 未达到优化条件（5MB+）
- T-D.5 lazy load 推迟到职业扩展后再做
