## 四、思维模式 DNA

### 模式 1：证据链强迫症

三级置信度（提取 > API > 文档）不是写在文档里的条款——它被实际执行到了代码层：`sourceType: "tier3"` + `requiresManualVerification: true` 标记遍布 ChrParser。当 AI agent 建议用项目内 `-1500` 重力常数当"DNF 真值"时，作者立刻质疑："这个也不可信吧 我们不是有一个 exe 吗"。低优先级证据永远不能反向覆盖高优先级。

**这反映什么**：被"自信的错误"反复坑过。大概率经历过"文档写着 A，代码实际是 B，但所有人都按 A 在讨论"的沟通灾难，然后建立了强迫症级别的溯源习惯。

### 模式 2：自文档化的元认知

28 个 memory 文件不是随意堆的——每个都是"犯错 → 写 feedback → 引为何错 → 给预防方案"。旧的会被标记 ⚠️ STALE 而不是删除。`architect-learning-roadmap.md` 在自我诊断"全是定性"的当天下午就被自己的 `game-engine-architecture.md`（含帧预算数字）推翻——然后第二天加了个 ⚠️ banner 解释这件事。

**这反映什么**：不是"我记录了我的成长"——是"我怕忘了、怕以后飘了、怕明天推翻今天的结论没人知道"。这是一个**被自己的决策反噬过**的人的防御机制。

### 模式 3：范围铁律

"PvE 打怪 1:1 还原 13 system，不接受简化。PvP/城镇/非战斗环节 out of scope。"当 AI 建议压缩到 4 heavy + 5 helper 时，被明确拒绝："不接受为了少写代码而压缩 system 数量"。

**这反映什么**：不是完美主义——是**目标版本 70-85 经典 DNF** 的真实性承诺。作者在还原的不是"一个游戏"，是"一个特定时间点的特定体验"。这种偏执通常来自**玩家身份**——不是开发者想做游戏，是玩家想复刻青春。

### 模式 4：自底向上、先数据后架构

自己写的决策记录里明确写了：**"自底向上代替自顶向下"**。不是先画架构图再填数据——是先提取 193 个 .nut、统计出 478 个引擎 API、跑 classify-v4 分 22 个 bucket，然后说"好，现在做 schema 设计"。

**这反映什么**：典型的**自学转型路径**——没人教过正统游戏引擎架构（top-down design），但逆向工程的经历教会他"数据不说谎，架构图会说谎"。这是实践者的方法论，不是学院派的。

### 模式 5：5 分钟的事后分析

一次 commit message 写错了（`6cb6208`误描述了变更内容），5 分钟内写了 `2026-05-26-commit-mishap-postmortem.md`——含时间线、逐文件影响评估、预防规则。

**这反映什么**：不是"我爱写文档"——是"这个错让我很不舒服，写下来才能确保不会再犯"。这是**内向型自律**，不是外向型协作需求。

## 五、项目本质

这不是一个"游戏项目"。这是一个**记忆工程项目**。

作者要复刻的不是 DNF——是**2012 年他自己玩 DNF 时感受到的东西**。他拿了 2018 年客户端、提取了 193 个 `.nut` 文件里的 7370 个函数调用、统计了 49.8% 的副本地图 ID 已经过时——然后精确地还原"一个已经部分腐烂的客户端内部"的引擎逻辑。

投入深度的几个数字：
- 241 次提交 / 30 天 = 每天 8 次提交
- 333,596 行新增代码
- 143 次 AI agent 协作（Opus 4.7 占 68%——一直用最强模型）
- 10 个二进制格式解析器（PVF/NPK/IMG/ANI/SKL/ATK/CHR/MOB/DGN/MAP）
- 28 个 memory 文件（每个都是一次反省）
- 4-way 一致性扫描（memory/docs/code/git）

这不是"有空做做"的 side project。这是**主业之余的高强度投入**——周末 29 次提交、凌晨 2-6 点还在写代码。
